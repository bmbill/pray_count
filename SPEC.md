# 佛教功課紀錄 PWA — 功能規劃書 (SPEC)

> 目標使用者：不熟悉 3C 的長輩。介面需大字、溫馨、友善、步驟簡單，第一次有教學。

---

## 1. 技術架構

| 項目 | 選擇 |
|------|------|
| 前端 | PWA（可安裝到手機桌面）、大字體 UI、可離線瀏覽 |
| 前端框架 | Vite + React + TypeScript |
| 路由 | HashRouter（GitHub Pages 靜態空間相容，分享網址用 `#/...`） |
| 後端 / 資料庫 | Supabase（PostgreSQL + Auth + 即時同步，免費額度） |
| 部署 | GitHub Pages（GitHub Actions 自動建置部署） |
| 多語言 | 預設繁體中文（zh-TW），可切換英文（en） |

**分享網址範例**
`https://<帳號>.github.io/pray_count/#/join/<專案短碼>`

---

## 2. 身分與同步模型（無帳號密碼）

- 使用者**只需輸入姓名**即可開始。
- 每個「人」在資料庫是一筆 `users` 記錄，有一組隱藏的唯一 ID；姓名只是顯示用。
- 每個「裝置」啟動時透過 Supabase 匿名登入取得一組裝置金鑰（JWT），對應到某個「人」。
- **跨裝置同步（配對碼）**：
  1. 裝置 A →「設定 > 同步裝置」→ 產生一組短碼（例如 6 碼、10 分鐘內有效）。
  2. 裝置 B → 輸入該短碼 → B 這台裝置就綁定成「同一個人」，看到相同資料。
- 同組若有同名者，加入時會提示「已有相同姓名，確定是您本人嗎？」以避免混淆。

**安全模型**：以 Supabase Row Level Security（RLS）保護——每個人只能讀寫自己的紀錄；小組統計數字透過受控的資料庫函式（RPC）計算並依隱私設定回傳，不直接開放他人原始紀錄。

---

## 3. 角色與權限

| 角色 | 權限 |
|------|------|
| 組長（可多位共同管理） | 建立/編輯專案、新增/編輯/排序/停用功課項目、設定目標次數、開始/結束日期、隱私模式、管理組長名單、查看分享網址 |
| 成員 | 加入專案、記錄自己的次數、查看自己與小組統計 |

- 建立專案者自動成為組長；組長可把其他成員設為共同組長。

---

## 4. 專案（小組）與功課項目

**專案**
- 名稱、說明（可選）
- 分享短碼 / 網址（自動產生）
- 開始日期、結束日期（結束日期可不設定＝長期進行）
- 隱私模式（見下）

**功課項目（僅組長可編輯）**
- 名稱（自由輸入，如：般若波羅蜜多心經、度母咒、密集嘛…）
- 目標次數（可不設定；有設定時畫面顯示「整組累計 / 目標」進度條）
- 排序、可停用（保留歷史紀錄）

---

## 5. 記錄操作方式

主畫面 = 該專案的功課項目清單，每個項目一張大卡片：

```
┌─────────────────────────────┐
│  般若波羅蜜多心經              │
│  今天：36    累計：1,208       │
│  [ -100 ][ -10 ][ -1 ] [ +1 ][ +10 ][ +100 ] │
│  ▓▓▓▓▓▓░░░░  整組 12,080 / 100,000 │  ← 有設定目標時才顯示
└─────────────────────────────┘
```

- 六個大按鈕：`-100  -10  -1  +1  +10  +100`
- 每次點擊寫入一筆「增減紀錄」（delta），可為負數（更正用），保留完整歷史。
- 即時顯示「今天」與「累計」。

---

## 6. 隱私模式（組長選擇）

| 模式 | 說明 | 預設 |
|------|------|------|
| 僅總計 | 成員只看自己的紀錄 + 小組總計 | ✅ 預設 |
| 公開前三名 | 額外顯示排行榜前 3 名 | |
| 完全公開 | 顯示所有成員各自累計 | |

---

## 7. 統計功能

**個人（每個專案內）**
- 每日 / 每週 / 每月 / 全部累計 / 自訂區間
- 分項目與全部合計

**小組整體**
- 每日 / 每週 / 每月 / 全部累計 / 自訂區間
- 各功課目標進度（有設目標時）
- 排行榜（依隱私模式）

**個人總覽頁（跨所有專案）**
- 我參與過的所有小組清單
- 每個小組 / 每項功課的完成狀況（我的累計、小組累計、目標、進度）

> 時間分界（日/週/月）以裝置所在時區為準，記錄當下即存下「當地日期」，避免跨時區誤差。

---

## 8. 設定頁

- **語言**：繁體中文（預設）/ English
- **字體大小**：小 / 中 / 大 / 特大
- **字型**：提供 2–3 種易讀字型
- **佈景 / 介面主題**：溫馨配色
- **同步裝置**：產生配對碼 / 輸入配對碼
- 設定會隨「人」同步到其他已配對裝置。

---

## 9. 第一次使用教學

- 首次開啟：簡短輪播教學（3–4 頁圖文）→ 輸入姓名 → 進入。
- 之後可於設定頁「重看教學」。

## 10. 資料表設計（Supabase / PostgreSQL）

```
users            id, display_name, settings(jsonb: lang/font_size/font/theme), created_at
user_devices     device_uid(auth.uid) → user_id                 -- 一個人可多台裝置
pairing_codes    code, user_id, expires_at, created_at

projects         id, name, description, share_slug(unique),
                 start_date, end_date(nullable), privacy_mode,
                 created_by, created_at
project_leaders  project_id, user_id                             -- 多位組長
project_members  project_id, user_id, joined_at

items            id, project_id, name, target_count(nullable),
                 sort_order, is_active, created_at

records          id, item_id, project_id, user_id,
                 delta(int, 可負), recorded_at(timestamptz),
                 record_date(date, 當地日期)
```

**統計以 RPC 函式計算**（依區間 sum(delta) 分組），並在函式內依 `privacy_mode` 控管可回傳的他人資料。

---

## 11. 開發階段規劃

1. **Phase 1 — 基礎**：Supabase 專案建立、資料表 + RLS、前端骨架、匿名登入、輸入姓名。
2. **Phase 2 — 核心**：建立專案 / 分享網址 / 加入、功課項目管理、記錄（±按鈕）。
3. **Phase 3 — 統計**：個人 / 小組 各區間統計、目標進度、排行榜、個人總覽頁。
4. **Phase 4 — 體驗**：多語言、字體/主題設定、配對碼同步、首次教學、PWA 安裝 + 離線。
